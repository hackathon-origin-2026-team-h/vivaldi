# vivaldi 要件定義書
## 1. 概要
### 1.1 プロジェクトの目的
本プロジェクトの目的は、学会発表などのイベントで利用する、文字起こしアプリケーションを開発することである。文字起こししたスピーチ内容を動的に、聴講者にとって理解しやすい形に噛み砕き、聴講者の理解を助ける。

## 2. 利用者の定義
本システムの利用者は、以下の2つのロールに分類される。

ロール|説明
-----|----
発表者|発表を行う者。トークセッションの作成と自身の音声の書き起こしを行う。
聴講者|発表を聞く者。書き起こされた音声を受信し、それを自分にとって理解しやすい文章に噛み砕き表示する。

## 3. 機能要件
### 3.0 画面一覧
- 発表者ビュー
- 聴講者ビュー

### 3.1 発表者ビュー
聴講者が使うために必要な開始機能。

- 発表者はトークセッションを作成することができる。
  - 各トークセッションは一意のsnowflakeを持つ
  - 発表者ビューで大きく聴講者ビューへのQRコードとUrlを表示する
  - 各トークセッションは発表前、発表中、発表後の3つのステータスを持つ
    - 開始ボタンを押すことで発表中に、もう一度押すことで発表後に遷移する
  - 各トークセッションは発表者のスピーチ内容を書き起こしたものを保持する

### 3.2 聴講者ビュー

- 聴講者は発表者が提示したQRコードをスキャンすることで聴講者ビューを開ける
  - 聴講者ビューのUrlは`/<トークセッションid>`
- 聴講者ビューでは、発表者の発言内容と、それを個人向けにわかりやすく意訳したものが表示される
  - リアルタイムに更新される
  - 個人向けにわかりやすく意訳する機能の詳細については、後で言及する
  - 1つの文章に対して1つの意訳が生成され、その右にあるボタンを押すことで、その意訳がわかりづらかったことを表明できる。

### 3.3 個人向けにわかりやすく意訳する機能

-  1つの文章に対して1つの意訳が生成される
-  文中の理解が難しい箇所(単語を想定)を、適切に言い変え、平易に文章を生成する
-  "理解が難しい"、は人およびコンテキストによって変化するため、スマートな意訳を行う必要がある
   -  「スマートな意訳」機能
      -  ユーザーが意訳に対してわかりづらいという表明をすることができる。そこで、その何がわかりづらかったかを思考し、記録する
         -  例えば、ユーザーはこの分野に対する専門性がないのではないか、など
      -  ユーザーの特性についてわかったことはlocalStorageに保存し、別トークセッションでも扱えるようにする
      -  実際の意訳を実行するときには、ユーザーの特性を踏まえた上で実行する

## 4. 非機能要件
システムの品質に関する要件

- ユーザービリティ
  - 始めて使う人でも、QRコードを読みとるだけですぐに聴講できること
  - 発表者は現在の状態が常に視覚的に明確であること
- パフォーマンス
  - 書き起こし、意訳はできるだけリアルタイムに行えるよう努力すること
- 信頼性
  - 発表中にシステムが停止しないこと
  - 万が一、ネットワークが瞬断しても、状態をサーバー側で適切に保持し、復帰できるようにすること
  - 書き起こしは、LLMを用いて噛んでいる箇所やフィラーを除去すること
- 拡張性
  - 聴講者の増減に柔軟に対応できること
